3. Demo Scenarios

Scenario A: End-to-End Cash Deposit (Create → Submit → Approve)

Demonstrates the full lifecycle from form creation by a teller through supervisor approval.

Part 1 — Teller Creates & Submits (login as teller1)

  1. Login as teller1 / demo123. You'll land on the Dashboard showing recent submissions and quick stats.
  2. Tap "New Form" (the ＋ button or "New Form" action). Select the "Cash Deposit" template from the list.
  3. Fill in the form fields:
    Customer NameSarah Wambui
    Account Number0112345678
    ID Number28456123
    Deposit Amount350,000
    CurrencyKES
    Source of FundsBusiness Revenue
    Denomination Breakdown1000×200, 500×100, 200×250
  4. Tap "Submit". The system generates a reference number (e.g., CD-20260312-XXXXXX) and routes it for approval. A confirmation screen with the reference number appears.
  5. Check Dashboard — the form now appears under "Recent Submissions" with status Pending Approval.

Part 2 — Supervisor Approves (login as supervisor1)

  1. Login as supervisor1 / demo123. Navigate to the Approvals screen.
  2. In the "My Queue" tab, you'll see the cash deposit from Sarah Wambui. Notice the claim status shows "Unclaimed".
  3. Tap "Pick Up" to claim the form. The button changes to "Review" and other supervisors will see "Claimed by supervisor1".
  4. Tap "Review" to open the form details. Review the customer information, amount, and denomination breakdown.
  5. Tap "Approve", add an optional comment (e.g., "Verified ID, funds source confirmed"), and confirm.
  6. The form status changes to Approved. Check the "My Items" tab to see it in your action history.

Scenario B: High-Value Multi-Tier Approval

Demonstrates the 2-tier approval chain triggered by amounts over KES 1,000,000.

Pre-loaded: The Wanjiku Enterprises deposit (KES 1,500,000, ref: CD-20260312-F01BDA) is already submitted and waiting for Tier 1 approval.

Tier 1 — Supervisor Approval

  1. Login as supervisor1. Go to Approvals → My Queue.
  2. Find the Wanjiku Enterprises deposit (KES 1,500,000). Notice the 2-Tier indicator.
  3. Pick Up the form, review the details, then Approve with comment: "Tier 1 verified — large deposit, escalating to Branch Manager."
  4. The status changes to Pending Tier 2. The form is now in the Branch Manager's queue.

Tier 2 — Manager Approval

  1. Login as manager1. Go to Approvals → My Queue.
  2. The Wanjiku Enterprises deposit now appears in your queue (it wasn't visible before Tier 1 was complete).
  3. Pick Up, review (including the Tier 1 approval from supervisor1 in the timeline), then Approve.
  4. Final status: Approved. The approval timeline shows both tiers with timestamps and comments.

Scenario C: Creating a New Form Template (Admin)

Demonstrates how an admin configures new form types for the organization.

  1. Login as admin1. Navigate to Templates from the bottom navigation.
  2. Tap "Create Template". Enter the template name: "Account Opening — Savings". Set Journey Type to ACCOUNT_OPENING.
  3. Add form fields using the field builder:
    Customer Full NameText field, required
    National ID NumberNumber field, required
    Date of BirthDate field, required
    Phone NumberPhone field, required
    Email AddressEmail field, optional
    Initial Deposit AmountCurrency field, required
    KRA PINText field, required
    Next of Kin NameText field, required
    Next of Kin PhonePhone field, required
  4. Save the template. It's initially in Draft status.
  5. Tap "Publish" to make it available to all tellers. Status changes to Published.
  6. Switch to teller1 and verify the new template appears in the "New Form" template list.

Scenario D: Admin Features Tour

Quick walkthrough of admin capabilities.

  1. Login as admin1. Go to Admin panel from bottom navigation.
  2. Users tab — View all system users, their roles, and active status. Shows branch assignments.
  3. Roles tab — View the 7 role mappings (e.g., "Branch Teller" maps to system role MAKER with permissions like FORM_CREATE, FORM_SUBMIT). Each card shows the role's color, permissions as chips, and scope.
  4. Rules tab — Tap to open Workflow Configuration. See approval rules organized by journey type (Cash Deposit, Cash Withdrawal, etc.). Each rule shows amount thresholds and tier configuration. Filter by journey type using the pills at the top.
  5. Themes tab — View available themes. The active "Coral Bank" theme shows the brand colors. You can create additional themes for white-labeling.
  6. Toggle dark mode using the theme switcher to show the adaptive theming — the entire UI adjusts with Deep Navy backgrounds and coral accents.

4. Search & Cross-Role Features

Historical Search

  1. From the Approvals screen (as any supervisor/manager), tap the "Search" tab.
  2. Try searching by reference number: type CD-2026 to find all cash deposits from 2026.
  3. Use filter chips: select "Cash Deposit" journey type, set status to "Approved", and optionally set a date range.
  4. Results appear as cards showing the form summary, status, and submission date. Tap any card to view the full form and approval timeline.

My Items — Track Your Activity

  1. From the Approvals screen, tap the "My Items" tab.
  2. This shows all forms you've either created or actioned (approved/rejected/returned).
  3. Each item shows your relationship: "You submitted", "You approved at Tier 1", etc.
  4. Great for tracking your daily activity and finding forms you've previously worked on.

5. Key Talking Points

Feature Business Value
Role-Based Queue Supervisors only see items they're authorized to approve — no confusion about who handles what. Reduces errors and speeds up processing.
Claim/Pickup Prevents duplicate work — when one supervisor claims a form, others see it's taken. Claims auto-expire after 15 minutes if not actioned.
Multi-Tier Approval High-value transactions automatically require additional approval layers. Configurable thresholds per transaction type — no code changes needed.
Approval Timeline Complete audit trail showing who approved what, when, and with what comments. Essential for compliance and regulatory requirements.
Configurable Workflow Rules Business users (admins) can adjust approval thresholds and tier requirements without developer involvement. Add new journey types on the fly.
White-Label Theming Fully branded experience with Coral Bank identity. Light and dark mode support. Themeable for different bank clients from the admin panel.
Dynamic Form Templates Create any form type through the admin UI — no coding required. Supports text, number, currency, date, phone, email, dropdown, checkbox fields.
Offline-Ready Architecture React Native foundation supports future offline mode for branch locations with intermittent connectivity.
